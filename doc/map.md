# 힐리힐리 모험지도 / 미션지도

내용 및 기능 요구사항

---

## 1. 목적

힐리힐리의 지도는 단순 길 안내용이 아니라, 함께 걷는 경험을 더 쉽게 이해하고 더 재미있게 참여하게 만드는 콘텐츠를 제공해야 한다.

초기에는 아래 2가지 지도 타입만 운영한다.

- 모험지도
- 미션지도

탐험지도는 현재 범위에서 제외한다.

또한, 사용자 위치를 실시간으로 수집하거나 GPX를 추적 저장하지 않는다.

---

## 2. 지도 타입 정의

### 2.1 모험지도

- **정의**: 함께 걷기 전에 길과 흐름을 이해하는 기본 지도
- **목적**
  - 코스와 집결지를 한눈에 이해하게 한다
  - 초보자도 참여 판단을 쉽게 하게 한다
  - 모임의 신뢰도와 준비도를 높인다
- **콘텐츠 요소**
  - 집결지
  - 시작점 / 종료점
  - 주요 포인트
  - 코스 요약
  - 거리 / 예상 시간 / 난이도
  - 지도 설명 / 코스 설명
  - 후기 연결
  - 이 지도로 모임 만들기

---

### 2.2 미션지도

- **정의**: 걸으면서 하나씩 달성하는 목표형 지도
- **목적**
  - 걷는 과정에 작은 목표와 재미를 제공한다
  - 단순 산행을 완료감 있는 경험으로 바꾼다
  - 국가숲길 콘텐츠를 더 기억에 남게 만든다
- **콘텐츠 요소**
  - 미션 포인트 2~5개
  - 특정 포인트 지나기
  - 스탬프 획득 구조
  - 포인트별 설명 / 힌트
  - 자유 방문형 또는 순서형
  - 완료 후 후기 작성 유도
  - 이 지도로 모임 만들기

---

## 3. 공통 요구사항

### 3.1 공통 콘텐츠 요구사항

모험지도와 미션지도 모두 아래 정보를 가질 수 있어야 한다.

- 지도 제목
- 산 이름
- 지역
- 지도 타입
- 요약 설명
- 거리
- 예상 시간
- 난이도
- 대표 이미지 또는 지도 썸네일
- 체크포인트/포인트 목록
- 후기 연결
- 모임 생성 연결

---

### 3.2 공통 기능 요구사항

모험지도와 미션지도 모두 아래 기능을 제공해야 한다.

- 트레일 목록에서 지도 타입 구분 표시
- 트레일 상세에서 지도 타입별 정보 표시
- 지도 화면에서 포인트 표시
- 체크포인트 목록 조회
- 이 지도로 모임 만들기
- 모임 상세에서 연결된 지도 정보 확인
- 후기와 지도 연결

---

### 3.3 비포함 요구사항

현재 범위에서 제외한다.

- 실시간 위치 공유
- 사용자 GPX 수집
- 백그라운드 위치 추적 기반 완주 판정
- 자동 포인트 도착 판정
- 게임형 배지/리워드
- 숨은 포인트/보물찾기
- 탐험지도

---

## 4. 모험지도 기능 요구사항

### 4.1 조회 기능

- 트레일 목록에서 모험지도 뱃지를 표시할 수 있어야 한다
- 트레일 상세에서 아래 정보를 볼 수 있어야 한다
  - 집결지
  - 시작점 / 종료점
  - 주요 포인트
  - 코스 요약
  - 거리 / 시간 / 난이도
  - 설명
- 지도 화면에서 주요 포인트가 보여야 한다

### 4.2 생성/관리 기능

운영자 또는 콘텐츠 등록자가 아래를 등록할 수 있어야 한다.

- 지도 기본 정보 입력
- 집결지 지정
- 시작점 / 종료점 지정
- 주요 포인트 추가/수정/삭제
- 포인트 순서 지정
- 코스 요약 입력
- 거리 / 시간 / 난이도 입력

### 4.3 모임 연결 기능

- 트레일 상세에서 이 지도로 모임 만들기가 가능해야 한다
- `CreateSessionScreen`에 아래 정보가 전달되어야 한다
  - `mountainName`
  - `routeName`
  - `routeSummary`
  - `trailId`
- 세션 상세에서 연결된 모험지도 정보를 확인할 수 있어야 한다

---

## 5. 미션지도 기능 요구사항

### 5.1 조회 기능

- 트레일 목록에서 미션지도 뱃지를 표시할 수 있어야 한다
- 트레일 상세에서 아래 정보를 볼 수 있어야 한다
  - 미션 설명
  - 미션 포인트 목록
  - 포인트별 설명
  - 완료 방식 안내
- 지도 화면에서 미션 포인트가 강조되어 보여야 한다

### 5.2 생성/관리 기능

운영자 또는 콘텐츠 등록자가 아래를 등록할 수 있어야 한다.

- 지도 기본 정보 입력
- 미션 설명 입력
- 미션 포인트 추가/수정/삭제
- 포인트별 설명/힌트 입력
- 방문 순서 지정 여부 설정
- 필수 포인트 여부 설정
- 완료 조건 설명 입력

### 5.3 모임 연결 기능

- 트레일 상세에서 이 지도로 모임 만들기가 가능해야 한다
- 세션 생성 후, 세션 상세/모험방에서 미션 포인트를 다시 볼 수 있어야 한다
- 후기 작성 시 미션 경험을 회상할 수 있도록 연결할 수 있어야 한다

---

## 6. 포인트(체크포인트) 요구사항

포인트는 지도 타입에 따라 다르게 사용한다.

### 6.1 포인트 유형

아래 정도로 구분 가능해야 한다.

- 집결지
- 시작점
- 종료점
- 일반 포인트
- 미션 포인트

### 6.2 포인트 공통 속성

각 포인트는 최소 아래 정보를 가진다.

- 제목
- 설명
- 위도
- 경도
- 포인트 타입
- 정렬 순서

### 6.3 미션지도 추가 속성

미션지도 포인트는 추가로 아래 속성이 있으면 좋다.

- 필수 여부
- 순서 방문 여부
- 힌트 또는 안내 문구

---

## 7. 트레일 탭 요구사항

### 7.1 목록 화면

- 전체 지도 / 내 지도 구조 유지
- 검색 가능해야 함
- 카드에서 지도 타입이 보여야 함
- 모험지도 / 미션지도를 구분해 볼 수 있어야 함

### 7.2 상세 화면

- 지도 타입에 따라 정보 영역이 달라져야 함
- 모험지도면 코스 이해 중심 정보 노출
- 미션지도면 미션 포인트와 완료 방식 중심 정보 노출
- 후기와 모임 만들기 액션이 보여야 함

### 7.3 지도 화면

- 모험지도: 집결지/시작/종료/포인트 중심 표시
- 미션지도: 미션 포인트 중심 표시
- 실시간 사용자 위치 추적 없이도 이용 가능해야 함

---

## 8. 모험 탭 연결 요구사항

- 지도 없이도 모임은 만들 수 있어야 한다
- 지도와 연결된 모임은 더 풍부한 정보가 보이도록 해야 한다
- 세션 상세에서 연결된 지도 정보를 보여줄 수 있어야 한다
- 미션지도 기반 모임이면, 세션 상세에서 미션 포인트 목록을 확인할 수 있어야 한다

---

## 9. 후기 연결 요구사항

- 후기 작성은 기존 방식 유지
- 후기 상세 또는 세션 상세에서 연결된 지도 정보를 볼 수 있어야 한다
- 추후 확장 시 “어떤 포인트가 좋았는지”를 후기와 연결할 수 있는 구조를 고려한다

---

## 10. 운영 우선순위

### 1순위

- 모험지도
- 미션지도 기본 타입 구분
- 포인트 등록/표시
- 트레일 상세 표시
- 모임 만들기 연결

### 2순위

- 미션 포인트 설명 강화
- 순서형 / 자유형 미션 설정
- 세션 상세 내 미션 포인트 요약

### 후순위

- 숨은 포인트
- 보물찾기
- 탐험지도
- 배지/리워드
- 자동 위치 인증

---

## 11. 한 줄 정리

- 모험지도: 길과 흐름을 이해하는 기본 지도
- 미션지도: 포인트를 따라 목표를 달성하는 지도
- 공통 핵심 가치: 지도를 통해 모임을 더 잘 이해하고, 더 재미있게 참여하게 만든다
